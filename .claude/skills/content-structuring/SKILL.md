# Content Structuring Skill（思考変革型）

目的:
- CSOのイシューを「読者の思考が変わる構造」に変換する
- 情報を届けるのではなく、読者の世界の見え方を変える設計をする
- 読んだ後に「なるほど、そういうことか」と「で、自分はどうするか」が両方起きる状態を作る

---

## 基本構造: 思考変革3層フォーマット

```
LAYER 1: 何が起きているか（現象の提示）
├ 100字以内
├ 読者が「確かにそれは知っている／感じている」と思える事実を出発点にする
├ 出典名・URL・公開日を明記する
└ 「驚かせる」のではなく「共鳴させる」入口にする

LAYER 2: なぜそれが起きているのか（構造とインサイト）
├ 300〜500字
├ 表層の現象を1〜2段掘り下げ、構造的な理由を説明する
├ 最後に「中心にある人間のインサイト」を1文で言語化する
├ 「海外の話」ではなく「人間の普遍的な話」として書く
└ 読者が「これは自分にも当てはまる」と感じる抽象度にする

LAYER 3: 思考と行動の変化
├ 「見方が変わること」を先に提示する（思考の変化）
├ その上で「明日からできること」を1〜2点に絞る（行動の変化）
├ 3択判定を入れる: 試す価値あり / まだ早い / 不要
└ 「知識を得た」ではなく「自分が変わった」と感じさせる終わり方にする
```

設計の原則:
- **LAYER 1は「共鳴」の入口**。驚かせるより「そうそう、それ」と思わせる。
- **LAYER 2は「世界観の更新」**。知識ではなく、ものの見方が変わる体験を作る。
- **LAYER 3は「自分ごとへの変換」**。行動より先に思考の変化を示す。

---

## 出典明示のルール（必須）

- LAYER 1には必ず出典（名称＋URL＋公開日）を記載する
- 「海外で話題」「最新研究によれば」などの曖昧な表現は使わない
- CSOが付与した信頼スコアが5点未満の情報は使用しない

---

## タイトル・フック設計のルール

タイトル（5案以上）:
- 「問い」の形にする。読者が「考えさせられる」入口を作る
- 型の例:「なぜ〇〇は□□になったのか」「〇〇の時代に、□□する理由」
- NGパターン: 「〇〇の方法」「〇〇を上司に説得する」「〇〇の最新トレンド」

冒頭フック（3案）:
- 読者がすでに感じている「違和感・不安・疑問」から入る
- LAYER 1の現象を「問い」として投げかける形にする
- 「答えを教える」のではなく「一緒に考えたくなる」入口にする

---

## チャネル別の展開ルール

| チャネル | 重点レイヤー | 形式 |
|---|---|---|
| X投稿 | LAYER 1 ＋ LAYER 2の核心1文 | 「なぜ〇〇か」という問い＋1文の答え |
| note/ブログ | LAYER 1〜3フル | 800〜1500字。問いで始まり、思考の変化で終わる |
| YouTube/TikTok | LAYER 2中心 | 「あなたが知らなかった構造」として口頭で展開 |

---

## CPOへの引き渡し基準

以下が揃っている状態で渡す:
- 3層の各レイヤーに内容が埋まっている
- LAYER 2にインサイト（人間の本質）が1文で入っている
- LAYER 3に「思考の変化」が先に書かれている
- 出典が明記されている
- 3択判定と具体行動が書かれている
- タイトル案（問いの形）が5本以上ある
