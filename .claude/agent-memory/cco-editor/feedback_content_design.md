name: コンテンツ設計の原則（フィードバック）
description: 思考変革型コンテンツの設計で守るべきルールと避けるべきパターン
type: feedback
---

**タイトルは「問いの形」にする。**
Why: 「〇〇の方法」「最新トレンド」型は情報収集目的の読者にしかリーチせず、AIで代替される。読者が「考えさせられる」入口を作ることがブランドの差別化になる。
How to apply: タイトル案を出すときは全案を「なぜ〇〇か」「〇〇の時代に□□する理由」などの問い形式にする。NGパターン（「〇〇の方法」「〇〇のトレンド」）は出さない。

**LAYER 3は思考の変化を先に書き、行動は後に置く。**
Why: 行動を先に出すとハウツー記事になり、ブランド原則（構造を伝える）から外れる。
How to apply: LAYER 3の冒頭は必ず「見方が変わること」から始める。

**LAYER 2のインサイトは「海外の話」ではなく「人間の普遍的な話」として書く。**
Why: 読者が「これは自分の話だ」と感じなければ、世界観の更新が起きない。
How to apply: インサイトを1文に圧縮するとき、主語を「マーケターは」ではなく「人間は」に置き換えられるかチェックする。

**出典は曖昧な表現を使わない。**
Why: 「海外で話題」「最新研究によれば」はブランドの信頼性を下げる。
How to apply: LAYER 1には必ず「名称・URL・公開日」の形式で出典を明記する。信頼スコア5点未満の情報は使用しない。
