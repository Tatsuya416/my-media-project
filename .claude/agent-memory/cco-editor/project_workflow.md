description: CSO→CCO→CPOの引き渡しフローと各役割の責任範囲
type: project
---

CSO（戦略）→ CCO（構成設計）→ CPO（制作）という順序でコンテンツが流れる。

**CCOの責任範囲:**
- CSOのイシューをコンテンツ構成に変換する
- タイトル5案・冒頭フック3案・3層構成・チャネル別展開・CTAを出力する
- CPOがそのまま制作できる粒度まで落とす（引き渡し基準を満たすこと）

**CPOへの引き渡し基準（必須チェックリスト）:**
- 3層フォーマット（LAYER 1〜3）すべてに内容が埋まっている
- LAYER 2にインサイト（人間の本質）が1文で入っている
- LAYER 3に「思考の変化」が行動より先に書かれている
- 出典（名称・URL・公開日）が明記されている
- 3択判定と具体的な行動が書かれている
- タイトル案（問いの形）が5本以上ある

**Why:** 役割間の越権を防ぎ、各担当が自分の範囲に集中できる構造にするため。
**How to apply:** CCOとして動くときは、CPOの仕事（文章の肉付け・実際の執筆）には踏み込まず、構成の骨格を渡す粒度を維持する。
